BAB 6 : QUALITY ASSURANCE
- Kalau semua sudah selesai, maka tugas kita sekarang adalah sebagai QA, yang tadinya jadi PM / Lead dari AI, sekarang jadi QA juga, kita test seluruh flow, dari login, sampai seluruh CRUD kita test, ya karena kita mau bikin production apps jadi harus di cek jangan sampai ada bug

- Misal ni ada Create & edit belum berfungsi dengan baik, bilang aja ke agentnya, Periksa view user management, di create dan edit belum berfungsi, tolong cek di controller, model, dan migration, sinkronkan agar bisa create dan edit

Prosesnya secepat apa? 1-3 menit untuk ini semua dia cek seluruhnya

- Atau case lain nextjs, atau react, biasanya ada ratusan error typescript untuk MVP, di kisaran 500 - 1000, dari awal build sampai selesai, jadi ada trik khusus, maka ketika pembuatan, dia harus jalankan npm install typescript dulu, lalu suruh dia npx tsc --noEmit setelah selesai bikin 10 file, dan memastikan seluruh error typescript dibersihkan, jadi bener bener clean code

- Kasus lain kalau nyelesaikan 1000+ error type script juga gak butuh waktu lama dia, butuh sekitar 4x proses, dan masing2 proses makan waktu 5-10 menit, jadi kisaran setengah jam 1000 error type script itu hilang

- Ada 1 prompt khusus untuk QA
"Lakukan npx tsc --noEmit setiap selesai 10 task, dan buatlah Dokumen checking untuk mengecek VIEW, model, dan migration apakah sudah sinkron dan buat kesimpulan"

kalau sudah dapat kesmpulan dan ada yg belum sinkron "perbaiki", ini untuk memastikan seluruh CRUD berjalan dengan baik
